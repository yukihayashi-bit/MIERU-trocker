**ベース仕様:** SPECIFICATION.md (Web版)  
**技術スタック:** React Native (Expo) / expo-router / expo-sqlite / Supabase  
**作成日:** 2026-04-13

---

## 1. 設計方針

### 1.1 Web版との役割分担

| 機能 | Web版 (PC) | iOSアプリ |
|------|-----------|----------|
| 打刻（タイムトラッカー） | ○ | **主役** |
| 今日の自分の履歴 | ○ | ○ |
| ダッシュボード（全体分析） | **主役** | 簡易版（自分のサマリーのみ） |
| カテゴリ管理 | **主役** | 閲覧のみ |
| スタッフ/メンバー管理 | **主役** | なし |
| 新規病院登録 | **主役** | なし |
| データエクスポート | **主役**（将来） | なし |

### 1.2 モバイル特有の設計原則

1. **片手操作最適化** - 打刻ボタンは親指の届く画面下部に大きく配置
2. **オフラインファースト** - 全打刻データはまずSQLiteに保存、バックグラウンド同期
3. **最小限のナビゲーション** - 打刻に到達するまで0タップ（起動即打刻画面）
4. **同期状態の常時表示** - 未同期件数とオンライン/オフライン状態を常に視認可能

## 4. 画面仕様

### 4.1 ルートレイアウト (`app/_layout.tsx`)

**役割:** アプリ全体の初期化と認証ガード

**処理:**
1. `DatabaseProvider` で SQLite を初期化（テーブル作成・マイグレーション）
2. `AuthProvider` で expo-secure-store から認証トークンを復元
3. `SyncProvider` で同期エンジンを起動
4. 認証状態に応じて `(auth)` or `(tabs)` にルーティング

```
認証トークンあり → (tabs) へ（オフラインでもアプリ起動可能）
認証トークンなし → (auth)/login へ
```

### 4.2 ログイン画面 (`app/(auth)/login.tsx`)

**概要:** モバイル最適化したハイブリッド認証画面

**レイアウト（上から順）:**

```
┌─────────────────────────┐
│                         │
│      MIERU ロゴ         │
│      タイムスタディ      │
│                         │
│  ┌───────────────────┐  │
│  │ 病院コード (8桁)  │  │  ← 常時表示、大きめフォント
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │ スタッフID        │  │  ← プレースホルダー: "スタッフIDまたはメール"
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │ パスワード        │  │
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │    ログイン       │  │  ← 大型ボタン、タップしやすい高さ
│  └───────────────────┘  │
│                         │
│  前回のログイン情報を    │  ← 病院コードを記憶して自動入力
│  記憶する ○             │
│                         │
└─────────────────────────┘
```

**UIコンポーネント:**
- `HospitalCodeInput` - 病院コード入力（8桁、自動大文字変換なし）
- `StaffIdInput` - スタッフID or メール入力（`@`検知でメールモード切替）
- パスワード入力（SecureTextEntry）
- ログインボタン（大型、height: 56px以上）
- エラーメッセージ表示

**ロジック:**
- `@` を含む → メールアドレスとして認証（病院コード入力を薄くグレーアウト）
- `@` を含まない → スタッフID + 病院コードからダミーメール合成して認証
- 認証成功 → トークンを `expo-secure-store` に保存
- 認証成功 → ユーザー情報（id, tenant_id, role, name）を SQLite にキャッシュ
- 「病院コードを記憶する」ON → `expo-secure-store` に病院コードを保存し次回自動入力

**オフライン起動:**
- 前回ログイン成功時のトークンが `expo-secure-store` に残っていればログイン画面をスキップ
- トークン有効期限切れ → オンライン復帰時にリフレッシュ、失敗ならログイン画面へ

### 4.4 打刻画面 (`app/(tabs)/index.tsx`) - メイン画面

**概要:** アプリの主役。片手で素早く操作できるストップウォッチUI。

**レイアウト:**

```
┌─────────────────────────────┐
│ ● オンライン  未同期: 3件   │  ← SyncStatusBar
├─────────────────────────────┤
│                             │
│        00:23:45             │  ← StopwatchDisplay (大型デジタル)
│        経過時間             │
│                             │
│    開始: 14:30              │  ← 打刻開始時刻（計測中のみ）
│                             │
│                             │
│                             │
│                             │
│   ┌─────────────────────┐   │
│   │                     │   │
│   │      ■ 停止         │   │  ← PunchButton (直径120px)
│   │                     │   │     待機時は「▶ 開始」
│   └─────────────────────┘   │
│                             │
│  ─── 今日の記録 (5件) ───   │
│  14:00-14:30  バイタル測定  │  ← DayLogList（直近3件、コンパクト表示）
│  13:15-14:00  看護記録      │
│  11:30-13:00  清潔ケア      │
│            ▼ もっと見る     │
└─────────────────────────────┘
```

**UIコンポーネント:**

| コンポーネント | 役割 |
|--------------|------|
| `SyncStatusBar` | オンライン/オフライン状態 + 未同期件数（画面上部、高さ28px） |
| `StopwatchDisplay` | 経過時間の大型デジタル表示（フォントサイズ: 56px） |
| `PunchButton` | 開始/停止の大型円形ボタン（直径120px、画面下寄り配置） |
| `ActiveLogBanner` | 計測中の開始時刻と経過を表示 |
| `DayLogList` | 今日の完了済みログ（直近3件、省略表示） |
| `CategorySheet` | 停止時にボトムシートで表示されるカテゴリ選択UI |

**打刻フロー:**

```
1. ユーザーが「▶ 開始」をタップ
   → 触覚フィードバック (Haptics.impactAsync)
   → SQLite に time_log を INSERT (synced = false)
   → ストップウォッチ開始
   → ボタンが「■ 停止」に変化（赤色）

2. ユーザーが「■ 停止」をタップ
   → 触覚フィードバック
   → CategorySheet（ボトムシート）が下からスライドアップ

3. カテゴリを選択
   → SQLite の time_log を UPDATE (end_time, duration, category_id)
   → ストップウォッチリセット
   → ボタンが「▶ 開始」に復帰（青色）
   → バックグラウンドで Supabase に同期を試行
```

**CategorySheet（ボトムシート）の構成:**

```
┌─────────────────────────────┐
│ ── 業務カテゴリを選択 ──    │
│                             │
│ ▸ 直接ケア                  │  ← マスターカテゴリ（セクションヘッダー）
│   ● バイタル測定            │  ← テナントカテゴリ（色付きドット + 名前）
│   ● 清潔ケア               │
│   ● 与薬・注射             │
│   ● 食事介助               │
│   ● 排泄介助               │
│   ● 患者対応               │
│                             │
│ ▸ 間接ケア                  │
│   ● 移動・搬送             │
│   ● 巡視                   │
│                             │
│ ▸ 記録                      │
│   ...                       │
└─────────────────────────────┘
```

- マスターカテゴリ（Level 1）で `SectionList` のセクション分け
- `sort_order` 順に並び替え
- 各テナントカテゴリの左にカラードットを表示
- タップで即選択（確認ダイアログなし = スピード重視）

## 5. 同期ステータスUI仕様

### 5.1 SyncStatusBar（全画面共通ヘッダー下）

**表示パターン:**

| 状態 | 表示 | 色 |
|------|------|-----|
| オンライン & 全同期済み | `● オンライン` | 緑 (#22c55e) |
| オンライン & 未同期あり | `● オンライン　未同期: N件` | 緑 + オレンジバッジ |
| オンライン & 同期中 | `↻ 同期中...` | 青 (#3b82f6)、アニメーション |
| オフライン & 未同期あり | `○ オフライン　未同期: N件` | グレー (#9ca3af) + オレンジバッジ |
| オフライン & 全同期済み | `○ オフライン` | グレー |
| 同期エラー | `⚠ 同期エラー　タップしてリトライ` | 赤 (#ef4444) |

**高さ:** 28px（コンパクト、常時表示でも邪魔にならない）
**タップ動作:** エラー時はタップで再同期を試行

### 5.2 同期の動作タイミング

| トリガー | 動作 |
|---------|------|
| 打刻完了（カテゴリ選択後） | 即座に同期試行 |
| アプリフォアグラウンド復帰 | 未同期チェック → 同期試行 |
| ネットワーク復帰（オフライン→オンライン） | 全未同期データを一括同期 |
| 手動同期ボタンタップ | 即座に同期試行 |
| バックグラウンド（15分間隔） | expo-background-fetch で同期試行 |

## 6. ローカルDB (SQLite) スキーマ

### 6.1 テーブル定義

```sql
-- 打刻記録（メインテーブル）
CREATE TABLE time_logs (
  id            TEXT PRIMARY KEY,       -- UUID (ローカル生成)
  user_id       TEXT NOT NULL,
  tenant_id     TEXT NOT NULL,
  category_id   TEXT,                   -- 終了時に設定
  start_time    TEXT NOT NULL,          -- ISO 8601
  end_time      TEXT,
  duration_seconds INTEGER,
  created_at    TEXT NOT NULL,
  synced        INTEGER DEFAULT 0,      -- 0=未同期, 1=同期済み
  server_id     TEXT,                   -- Supabase 側のID（同期後に設定）
  sync_error    TEXT                    -- 最後の同期エラーメッセージ
);

-- カテゴリキャッシュ（サーバーからダウンロード）
CREATE TABLE categories_cache (
  id                TEXT PRIMARY KEY,
  master_category_id TEXT NOT NULL,
  master_name       TEXT NOT NULL,
  master_sort_order INTEGER NOT NULL,
  name              TEXT NOT NULL,
  color             TEXT NOT NULL,
  is_active         INTEGER DEFAULT 1,
  updated_at        TEXT NOT NULL       -- キャッシュ鮮度管理
);

-- 認証情報キャッシュ
CREATE TABLE auth_cache (
  key   TEXT PRIMARY KEY,              -- 'user_profile', 'tenant_info' など
  value TEXT NOT NULL,                 -- JSON
  updated_at TEXT NOT NULL
);
```

### 6.2 同期フロー

```
[打刻完了]
    │
    ▼
[SQLite INSERT (synced=0)]
    │
    ▼
[ネットワーク確認]
    │
    ├── オンライン → Supabase に INSERT
    │                   │
    │                   ├── 成功 → SQLite UPDATE (synced=1, server_id=xxx)
    │                   │
    │                   └── 失敗 → SQLite UPDATE (sync_error=message)
    │                              → 次回同期タイミングでリトライ
    │
    └── オフライン → 何もしない（次回オンライン時に同期）
```

## 7. 認証フロー（モバイル版）

```
[アプリ起動]
    │
    ▼
[expo-secure-store からトークン読み取り]
    │
    ├── トークンあり
    │     │
    │     ▼
    │   [SQLite auth_cache からユーザー情報復元]
    │     │
    │     ├── オンライン → Supabase でトークン検証
    │     │                  │
    │     │                  ├── 有効 → (tabs) へ遷移
    │     │                  └── 無効 → リフレッシュ試行
    │     │                              │
    │     │                              ├── 成功 → (tabs) へ
    │     │                              └── 失敗 → login へ
    │     │
    │     └── オフライン → (tabs) へ遷移（ローカルデータで動作）
    │
    └── トークンなし → login 画面へ
```

**重要:** オフライン時でもトークンが存在すれば打刻画面を開ける。打刻データはローカルに蓄積され、オンライン復帰時に一括同期される。
